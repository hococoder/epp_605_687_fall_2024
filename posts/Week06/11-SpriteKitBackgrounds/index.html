Week 6 - SpriteKit Backgrounds

Backgrounds

When you’re building a scene, you have 2 ways to render a background - you can either make it a single image, or you can use tiles. Using tiles provides you fine control of your scene’s background, and you can reference each tile if you want. Tiles also are more memory friendly - a continuous image may take a large amount of memory, depending on the size of your scene, but tiles can be reused as they fall off screen (think of how cells in a UITableView or pins in a MapView are reused).

Adding a background with the Tile Editor

A game that is about a spaceship just doesn’t feel right until it looks like it is space! To do that, let’s setup a tile set to populate our background with a series of random images (see the video above for a demo of these steps).

  • Right click on the project folder, choose New File, and then scroll down and choose SpriteKit Tile Set, calling it StarTileSet.
  • When that opens up, in the lower left hand corner, add a new Grid Tile Set, and call it Stars.
  • With that set selected, again go to the add button and choose “New Single Tile Group”.
  • Open up the Library and go to the media tab. I’ve got the star field sets in there that I showed earlier
  • Grab the first one, and drop it onto the center area where it says “Tile”.
  • After that you can add variants of that image by dragging over images that are similar. SpriteKit will use these variants to give your background some variety when painting it. You can even go into the variant’s properties on the right hand pane and change the Placement Weight value to cause that tile to appear more often than the others by increasing its value.

OK, now let’s use these tiles. Here we're going to use code to render the background, but you can also use an .sks file to define the scene. Make the following method, and we'll go through it.

   func addBackground()
  {
    let map = SKNode()
    addChild(map)
    
    let tileSet = SKTileSet(named: "Stars")!
    let tileSize = CGSize(width: 75, height: 75)
    let columns = 16
    let rows = 10
    
    let star = tileSet.tileGroups.first { $0.name == "Star"}
    
    let space = SKTileMapNode(tileSet: tileSet, columns: columns, rows: rows, tileSize: tileSize)
    space.fill(with: star)
    map.addChild(space)
  }

First, make a node and add it to the scene. Then grab the tileset we just made, define the tile size, and how many columns and rows for the background. Finally, make a SKTileMapNode and fill it with images from the "Star" group in the "Stars" set, and add that map node to the map.

Don't forget to add addBackground() to sceneDidLoad()

Run that in the simulator. The app looks great (if you see obvious tile boundaries in yours you can either eliminate tile variants that are too different from the original one you added, or make your tiles smaller so those changes in tiles are not so obvious).

This looks great, but let’s give some indication, other than the asteroids, that the Enterprise is moving in the next module.
